Ecosystem Overview

BickSpec Language / Compiler

A finance-oriented DSL compiled by the external Java/ANTLR project in bickspec-lang.

BickSpec Studio

Electron desktop IDE for editing, running, reviewing artifacts, and exporting reports.

VS Code Extension

Lightweight editor integration for .bks files, commands, artifacts, diagnostics, and setup.

Studio and the VS Code extension do not duplicate the compiler. Both use the real bickspec-compiler-1.0.0.jar built from bickspec-lang.

Language Reference

A program has zero or more IMPORT, FX, and FUNCTION declarations followed by exactly one PROJECT block.

IMPORT finance
FX GTQ := 7.80
FUNCTION margen(ingresos, costos) = (ingresos - costos) / ingresos
PROJECT "Example" {
  DISPLAY "Hello"
}
ConstructSyntax
ImportIMPORT ID
FX rateFX currency := numberLiteral
AssignmentID := expr
Batch assignmentid1, id2 := expr1, expr2
OutputDISPLAY expr or WRITE expr
InputREAD ID
ConditionalIF condition THEN ... ELSE ... END
LoopsWHILE condition DO ... END, REPEAT INT TIMES ... END
FunctionFUNCTION name(params...) = expr

Expressions, values, and comments

Expressions support arithmetic, comparison/equality, logical operators, parentheses, numbers, strings, identifiers, calls, money literals, time literals, and lowercase conversion/display suffixes such as to and in. Currencies include USD, GTQ, and EUR; time units include year, month, quarter, week, and day. Comments use # or /* ... */.

CAPEX := CAPEX_GTQ to USD
DISPLAY NPV_VAL in GTQ
MESES := MESES + 1 month

Lexer / token reference

ID, INT, FLOAT, strings, :=, =, arithmetic + - * /, comparisons >= <= == != > <, logical && || !, braces, parentheses, comma, keywords, currencies, and time units are tokenized by the grammar.

FX GTQ := 7.80
PROJECT "P3 - Input + IF" {
  DISPLAY "Ingrese tasa (0.xx):"
  READ R
  DISPLAY "Ingrese CAPEX en GTQ:"
  READ CAPEX_GTQ
  CAPEX := CAPEX_GTQ to USD
  IF R < 0.20 THEN
    DISPLAY "Tasa aceptable"
  ELSE
    DISPLAY "Tasa alta"
  END
  DISPLAY "CAPEX en USD:"
  DISPLAY CAPEX
}

BickSpec Studio

Studio is an Electron + React + TypeScript + Vite desktop IDE using Monaco Editor. It includes syntax highlighting, snippets, file explorer, tabs, recent projects, settings, a first-run setup wizard, linked compiler-repository configuration, result views, and report exports.

Unified Setup Guide

  1. Install Java; Java 21 is recommended.
  2. Install Maven to build the compiler JAR.
  3. Optionally install Graphviz for SVG parse-tree rendering.
  4. Build the compiler with mvn -f app/pom.xml package.
  5. In Studio or the VS Code extension, select or clone the bickspec-lang repository and validate the JAR.
  6. Run the setup test.

Reports and Exports

Report Preview uses real normalized run state. PDF, Excel, and CSV exports include summary data, runtime output or interactive transcript, diagnostics, artifacts, and build log. Export helpers remove the visual Program Output box and emit clean runtime lines; interactive exports use role/content rows.

Fases de Desarrollo

  1. Fase I — Analizador Léxico y Especificación Inicial: Definición formal del lenguaje .bks, reglas léxicas mediante expresiones regulares, gramática ANTLR4 inicial, tokens, palabras reservadas y casos de prueba base.
  2. Fase II — Parser, Árboles y Traducción Inicial: Implementación del parser, generación del parse tree, visitor semántico inicial, exportación DOT/SVG y primera traducción dirigida por la sintaxis hacia Java.
  3. Fase III — Compilador Funcional Completo: Tabla de símbolos, validación semántica, generación automática de Java, build con javac, ejecución del programa generado, artifacts estructurados y JAR final del compilador.
  4. Fase IV — Plugin de Visual Studio Code: Extensión ligera para VS Code con soporte para archivos .bks, syntax highlighting, snippets, comandos, output channel, diagnósticos, acceso a artifacts, terminal integrada para programas con READ y setup wizard.
  5. BickSpec Studio — Capa de Producto e IDE Desktop: Aplicación desktop construida con Electron, React, TypeScript y Vite para escribir, ejecutar, depurar, revisar artifacts y exportar reportes de especificaciones BickSpec usando el compilador real de bickspec-lang.

Troubleshooting

Java not found

Install Java or select a Java executable in setup. Java 21 is recommended.

Compiler JAR missing

Build from the repo with mvn -f app/pom.xml package.

Wrong repository folder selected

Select the root containing app/pom.xml and app/src, not a nested testing/output/target folder.

Graphviz missing

DOT output can still exist; SVG rendering may be unavailable until Graphviz is installed.

READ program needs input

Use interactive mode in Studio or the integrated terminal in VS Code.

Artifacts not found

Check build log, compiler output paths, and whether generation completed.

Diagnostics shown

Review Studio Errors or VS Code Problems for classified diagnostics and suggestions.

Unsigned installer warning

Windows may show SmartScreen; macOS may require right-click → Open during testing.
