Language Reference

Program Structure

Every `.bks` file follows a strict modular structure to ensure maintainability.

IMPORT common_math;

FX MyFinancialModule {
  PROJECT MyProject {
    money budget = 1000.00;

    main {
      IF (budget > 500) {
        PRINT("High liquidity");
      }
    }
  }
}

Money & Units

BickSpec uses a 2-decimal precision fixed-point internal model for USD.

  • Internal precision: 0.01
  • Time units: D, M, Y (Days, Months, Years)
  • Rates: Percentage based (0.05 = 5%)

Control Flow

Standard logic gates with financial focus:

IF (condition) { ... } ELSE { ... } WHILE (condition) { ... } REPEAT (times) { ... }

Diagnostics & Error Codes

SEM01

Undefined Symbol

Variable or function used without prior declaration in the current scope.

SEM03

Type Mismatch

Attempting to assign 'money' to a 'boolean' or 'string' variable.

GEN01

Code Generation Failure

Internal error during Java target source creation. Check file permissions.

EXEC04

Arithmetic Overflow

Financial calculation exceeded internal 64-bit precision limits.

Examples Gallery

Investment Payback

FX InvestmentAnalysis {
  PROJECT ROI_Tool {
    money initial_cost = 50000.00;
    money annual_benefit = 12000.00;

    main {
      money result = PAYBACK(initial_cost, annual_benefit);
      PRINT("Payback Period: " + result + " Years");
    }
  }
}

NPV Calculation

FX Valuation {
  PROJECT NPV_Calc {
    rate discount = 8%;
    money cfs = [100.00, 200.00, 300.00];

    main {
      money npv = NPV(discount, cfs);
      PRINT("Net Present Value: " + npv);
    }
  }
}
